# Tools Image Design Brief

This directory contains the canonical reference images for the weekly tools-report visual lane.

These images are intentionally **not** part of the main governance-house image idiom. The governance articles use a darker, more atmospheric "translation chamber" language about model interiority becoming accountable evidence. The tools-report lane is flatter, more diagrammatic, and more operational.

## Purpose

Use this lane for weekly tools coverage about spec-driven development, LLM/human collaboration, governed agent workflows, release coordination, and evidence-producing engineering surfaces.

The goal is to show tools as **shared coordination systems** rather than as brands, logos, dashboards, or anthropomorphic agents.

Treat these as the local visual reference set for this idiom.

## Exact Idiom

- Warm ivory paper ground with subtle texture
- Flat editorial geometry with minimal depth and almost no volumetric shading
- Dark charcoal outline drawing
- Muted teal routing lines representing coordination, handoffs, and agent flow
- Amber cards, tabs, folders, and modules representing specs, releases, or evidence packets
- Signal-red brackets, rails, bars, and gates representing review authority and interrupts
- Sparse symbolic interface elements rather than literal UI screenshots
- Strong negative space and poster-like composition

## Subject Matter

The scene should center on governed collaboration:

- specification cards
- release packets
- tool handoff paths
- review gates
- provenance rails
- folders, brackets, benches, houses, ladders, and routing junctions

The image should suggest staging, filtering, bundling, escalation, checkpointing, and shared control.

## Avoid

- No logos or brand marks
- No robots, faces, humanoids, or mascots
- No photoreal dashboards
- No neon sci-fi AI imagery
- No leaderboard graphics or trophy metaphors
- No readable interface text
- No drift back into the darker translation-chamber governance idiom unless the article is intentionally crossing lanes

## Agent Note

1. Read `docs/tools-image-style-guide.md`.
2. Read this file.
3. Select one seeded-random image from the canonical files in this directory as the style-continuity reference.
4. Use the guidance: `like this, but adjusted to the new article's narrative`.
5. Inspect the selected reference image and at least two other canonical files in this directory before writing prompts.
6. Keep the phrase `collaboration lattice` in the prompt.
7. Land new weekly tools assets in `/public/generated/tools/`.
